# Docs

## Purpose

`docs/` contains repository documentation, architecture notes, runbooks, changelogs, and agent plan/design records.

## Owner

The `sdkwork-news` repository owns documentation for news architecture and operations.

## Allowed Content

- Architecture decisions and design notes.
- Runbooks, changelogs, and migration notes.
- Superpowers plan/design records that reflect current standards.

## Forbidden Content

- Generated SDK transport output.
- Runtime state, secrets, or local override files.
- Duplicated copies of root SDKWork standards.
